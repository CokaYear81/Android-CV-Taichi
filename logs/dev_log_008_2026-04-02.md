# 开发日志 008 - week2 里程碑：训练友好的关键点 JSON 导出跑通

## 日期
- 2026-04-02

## 阶段
- 第 2 周 - 关键点采集与 JSON 导出

## 本次目标
- 在现有 `CameraX -> PoseLandmarker -> Overlay` 已跑通的基础上，增加一个最小可用采集流程
- 让用户能够在手机端输入：
  - `subject_id`
  - `action_name`
- 点击开始后，同时记录：
  - 原始视频 `mp4`
  - 训练友好的关键点 `json`
- 点击停止后，将样本保存到 App 私有目录，形成后续规则分析和 PyTorch 训练可直接使用的数据

## 本次完成内容

### 1. 新增最小采集 UI
- 在 App 底部新增采集控件：
  - `subject_id`
  - `action_name`
  - `Start Capture / Stop Capture`
- 去掉了 `front / left45 / right45 / side` 视角选项
- 当前版本默认按固定正面机位采集，不再额外记录 `view`

### 2. 新增视频录制能力
- 在原有 `Preview + ImageAnalysis` 基础上增加 `CameraX VideoCapture`
- 开始采集时同步开始录制视频
- 停止采集时结束录制
- 视频保存为：
  - `raw_videos/{sample_id}.mp4`

### 3. 新增训练友好的 JSON 导出
- 每段采集对应一个 JSON 文件
- JSON 顶层字段包含：
  - `sample_id`
  - `subject_id`
  - `action_name`
  - `capture_started_at`
  - `capture_ended_at`
  - `device_id`
  - `image_width`
  - `image_height`
  - `rotation_degrees`
  - `video_file`
  - `is_standard`
  - `error_tags`
  - `frames`
- 其中：
  - `is_standard` 先写 `null`
  - `error_tags` 先写空数组 `[]`

### 4. 每帧数据结构已固定
- `frames` 中每一帧都包含：
  - `frame_index`
  - `timestamp_ms`
  - `has_pose`
  - `pose_landmarks`
- `pose_landmarks` 固定为 `33` 个点
- 每个点固定包含：
  - `x`
  - `y`
  - `z`
  - `visibility`
- 当该帧没有检测到人体时：
  - 仍然保留该帧
  - `has_pose=false`
  - `pose_landmarks` 补齐为 `33` 个零点

### 5. 样本命名已固定
- 当前 `sample_id` 格式为：
  - `{subject_id}_{action_name}_{yyyyMMdd_HHmmss}`
- 不再包含视角字段

## 简短说明：文件保存在哪里
- 当前样本保存在手机 App 私有目录：

## 简短说明：如何把文件拉到电脑
- 打开 Android Studio：
  - `View -> Tool Windows -> Device Explorer`
- 进入：
  - `data/data/com.lenovo.taichivision/files/captures/`
- 找到目标 `json` 或 `mp4`
- 右键文件：
  - `Save As...`
  - 或 `Pull`
- 保存到电脑本地后，即可用于：
  - 数据检查
  - 标注准备
  - PyTorch 训练
