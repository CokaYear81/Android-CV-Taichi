# 开发日志 012 - `data_transform/trans.py` 切换到 `pose17_v1` 归一化口径

## 日期
- 2026-04-18

## 阶段
- `week4` 数据处理链路整理
- Android 导出格式与 Python 训练前处理口径对齐
- 文件路径优化

## 背景
- 当前安卓端正式导出格式已经是：
  - `landmark_schema_version = "pose17_v1"`
  - `frames[].pose_landmarks`（每帧固定 `17` 点）
- 旧版 `data_transform/trans.py` 仍按历史 `33` 点单帧口径处理：
  - 读取顶层 `landmarks`
  - 依赖 `33` 点索引（如 `23/24` 髋、`11/12` 肩）
- 该差异会导致新样本在归一化脚本阶段被误跳过，或输出数据结构不一致。

## 本次目标
- 将 `data_transform/trans.py` 统一到 `pose17_v1` 数据口径。
- 更新原本的动作文件存放路径为：（`DATA_DIR/<label>/*.json`）。
- 输出可直接进入训练前清洗/切分的帧级 CSV。

## 本次完成内容

### 1. 输入结构对齐到 `pose17_v1`
- 增加 schema 检查：
  - 仅处理 `landmark_schema_version == "pose17_v1"` 的样本。
- 读取逻辑改为多帧结构：
  - 从 `frames` 遍历每帧 `pose_landmarks`。

### 2. 归一化逻辑改为 17 点索引
- 采用 `pose17_v1` 索引定义：
  - 左肩 `1`、右肩 `2`
  - 左髋 `7`、右髋 `8`
- 归一化流程：
  - 以髋中心做中心化
  - 以肩中心到原点的距离做尺度归一化

### 3. 帧级过滤与输出
- 跳过无效帧：
  - `has_pose = false`
  - `pose_landmarks` 长度不是 `17`
  - 平均 `visibility` 低于阈值（默认 `0.5`）
- 输出改为帧级记录：
  - `filename`
  - `sample_id`
  - `frame_index`
  - `timestamp_ms`
  - `x0,y0,z0 ... x16,y16,z16`
  - `label`
- 默认输出文件保持：
  - `baduanjin_normalized.csv`

### 4. 动作文件路径存储优化
- 已经优化为data目录下按照不同的动作进行分类子文件夹

## 当前结论
- `data_transform/trans.py` 已与安卓端 `pose17_v1` 导出格式对齐。
- 当前数据链路已形成一致口径：
  - Android 导出 `pose17_v1`
  - Python 可视化工具读取 `pose17_v1`
  - Python 归一化脚本处理 `pose17_v1`
- 动作存储路径已优化

## 下一步建议

- 在训练前增加一层统计报告（每类样本数、平均时长、有效帧比例），作为数据质量基线。
