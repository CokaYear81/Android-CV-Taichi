# 开发日志 011 - `pose17_viz` 可视化包与 notebook 示例入口

## 日期
- 2026-04-09

## 阶段
- `week4` 辅助工具补充
- `pose17_v1` 数据检查工具整理期

## 背景
- 当前安卓端已经能稳定导出 `pose17_v1` 样本
- 为了更快检查动作样本质量、骨架连线和时间轴，需要一个本地可重复使用的 Python 可视化入口
- 之前根目录下的 notebook 更适合单次试验，不适合作为仓库内长期维护的统一入口

## 本次目标
- 将一次性的可视化 notebook 整理为可复用的 Python 包
- 在仓库内提供“只改文件名即可运行”的 notebook 入口
- 放入一个默认测试样本，降低首次使用门槛
- 修复可视化包首次导入时的语法错误

## 本次完成内容

### 1. 新增 `pose17_viz` Python 包
- 新增目录：`pose17_viz/`
- 当前提供的接口包括：
  - `load_sample(path)`
  - `summarize_sample(sample)`
  - `preview_frames(sample, rotation=...)`
  - `animate_sample(sample, rotation=..., interval_ms=..., frame_step=...)`
- 包内职责拆分为：
  - `io.py`：样本读取与 `pose17_v1` 校验
  - `transform.py`：显示方向变换
  - `plotting.py`：静态预览与骨架动图渲染

### 2. 新增 `data` 目录与 notebook 入口
- 新增目录：`data/`
- 新增 notebook：`data/pose17_viz_demo.ipynb`
- notebook 当前设计为：
  - 自动将 `android_app` 根目录加入 `sys.path`
  - 默认读取同目录下的 JSON 文件
  - 只需修改 `INPUT_JSON` 文件名，即可切换样本

### 3. 新增默认测试样本
- 新增样本：`data/0004_firstaction1_20260408_202501.json`
- 用于验证可视化链路是否能直接跑通

### 4. 修复导入报错
- 修复文件：`pose17_viz/io.py`
- 修复内容：
  - 更正 `f-string` 中 `landmark_schema_version` 的引号嵌套问题
- 修复原因：
  - 原始写法导致 notebook 在导入 `pose17_viz` 时直接触发 `SyntaxError`

## 当前使用方式
1. 打开：`data/pose17_viz_demo.ipynb`
2. 修改：`INPUT_JSON`
3. 运行 notebook 单元格
4. 查看：
   - 样本摘要
   - 首帧 / 中间帧 / 末帧预览
   - 右下角带时间标注的骨架动图

## 当前默认行为
- 默认显示方向为：`clockwise_90`
- 只对原始样本做可视化
- 不修改原始 JSON
- 不做时间重采样
- 不导出 `gif/mp4`
